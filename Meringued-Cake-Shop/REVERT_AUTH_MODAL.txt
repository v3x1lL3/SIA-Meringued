# Revert Login/Sign Up to previous GUI

To restore the previous single "Login" / "Sign Up" form (no "Continue as" step):

1. In index.html, replace the current auth modal (the block that starts with "<!-- Login/Signup Modal: \"Continue as\" ...") with the PREVIOUS MODAL HTML below.

2. Replace the "Modal Functions" script block (openAuthModal, openOrderModal, closeModal, authBackToContinueAs, authShowLogin, authShowSignup) with the PREVIOUS SCRIPT below.

--- PREVIOUS SCRIPT ---

        // Modal Functions
        function openAuthModal() {
            document.getElementById('authModal').classList.add('active');
            showLogin();
        }

        function openOrderModal() {
            document.getElementById('authModal').classList.add('active');
            showLogin();
        }

        function closeModal() {
            document.getElementById('authModal').classList.remove('active');
        }

        function showLogin() {
            document.getElementById('loginForm').style.display = 'block';
            document.getElementById('signupForm').style.display = 'none';
            document.getElementById('modalTitle').textContent = 'Login';
        }

        function showSignup() {
            document.getElementById('loginForm').style.display = 'none';
            document.getElementById('signupForm').style.display = 'block';
            document.getElementById('modalTitle').textContent = 'Sign Up';
        }

--- END ---

Note: authController.js and authModel.js still support role on signup; after revert you can leave them as-is (new signups will be customer by default) or remove the role parameter from signUpWithEmail if you prefer.
